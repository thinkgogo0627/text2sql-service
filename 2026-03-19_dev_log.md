## 해결한 버그 목록

| # | 에러 메시지 | 원인 | 해결 |
|---|---|---|---|
| 1 | `ModuleNotFoundError: No module named 'pipeline'` | `pipeline/` 디렉토리가 airflow 컨테이너에 마운트 안 됨 | docker-compose에 `../pipeline:/opt/airflow/pipeline` 볼륨 추가 |
| 2 | `TypeError: unsupported operand for \|: 'type' and 'NoneType'` | `int \| None` 타입 힌트는 Python 3.10+ 문법. Airflow 2.8.0은 Python 3.8 | `Optional[int]`, `Optional[datetime]` 으로 교체 |
| 3 | `AttributeError: 'Connection' object has no attribute 'commit'` | SQLAlchemy 1.4에서 `engine.connect()` 반환 객체는 `.commit()` 없음 | `engine.begin()` 사용 (자동 커밋/롤백) |
| 4 | `403 FORBIDDEN` 로그 서빙 에러 | webserver/scheduler 간 `secret_key` 불일치로 JWT 인증 실패 | 모든 airflow 서비스에 동일한 `AIRFLOW__WEBSERVER__SECRET_KEY` 설정 |
| 5 | `[Errno 13] Permission denied: 'data/raw/financials/...'` | `data/` 디렉토리 권한 부족 (airflow uid=50000) | `chmod -R 777 data/` + `_save_raw` try/except 처리 |
| 6 | 전체 extract 결과 `[]` (빈 데이터) | corp_code 다수 오류 + 지주사에 OFS 없음 | DART corpCode.xml 다운로드 후 stock_code로 전수 검증, CFS로 변경 |
| 7 | `필수 컬럼 누락: fs_div` | DART API 응답에 `fs_div` 컬럼이 없음 (요청 파라미터라 응답에 미포함) | `extract_dart_api()`에서 `df["fs_div"] = params["fs_div"]` 추가 |
| 8 | DBeaver 접속 불가 (`password authentication failed`) | 로컬 postgres가 `127.0.0.1:5432` 점유 → Docker 컨테이너 대신 로컬 DB로 연결됨 | Docker postgres 포트를 `5433`으로 변경 |
