```python
# transform_task
context["ti"].xcom_push(key="transformed_data", value=df_all.to_json(orient="records"))
# 61,083행 JSON = 수십 MB → Airflow XCom 한도(기본 48KB) 초과 → None 반환
```

- `load_task`: XCom pull → 파일 읽기 + 처리 완료 후 `os.remove()`
- `extract_task`도 동일 패턴 적용 (`raw_data.json`)

### BUG-02: 스테일 DAG 런 — `upsert_financials` 파일 없음으로 무한 루프

**현상**
- `scheduled__2025-10-01T00:00:00+00:00` 런의 `upsert_financials`가 attempt 2, 6 반복 실패
- 로그: `"변환 데이터 파일을 찾을 수 없습니다"`
- Airflow UI는 초록불

**원인 (3중 복합)**
1. 코드 수정 전에 `transform_and_load_financials`가 구 XCom 방식으로 이미 SUCCESS 마킹
2. Airflow는 retry 시 SUCCESS인 upstream 재실행 안 함 → 파일 미생성 상태로 `upsert_financials`만 재실행
3. 기존 코드는 파일 없을 때 `return` (침묵 성공) → SUCCESS 마킹됨에도 데이터 없음

**해결**
- 파일 없을 때 `return` → `raise FileNotFoundError(...)` 로 변경 (명시적 FAILED 처리)
- 해당 DAG 런 전체 Clear 후 재실행 필요

### BUG-03: corp_code leading zero 소실 → FK 위반

**현상**
```
ERROR: insert or update on table "financial_fact" violates foreign key constraint
DETAIL: Key (corp_code)=(266961) is not present in table "company_dim".
```
- `company_dim`에는 `"00266961"` (8자리), `financial_fact` INSERT 시도는 `266961` (6자리)

**원인**
```python
# pandas read_json dtype 자동 추론
pd.read_json('[{"corp_code": "00266961", ...}]')
# → corp_code 컬럼을 integer로 변환: 266961 (leading zero 제거)
# → str(266961) = "266961" (8자리 아님)
```

**해결**
```python
# pipeline/transform/financials.py:41
"corp_code": str(row.get("corp_code", "")).strip().zfill(8)

# pipeline/load/postgres_loader.py (2중 방어)
r["corp_code"] = str(r["corp_code"]).zfill(8)
```

